The 2 nonexistent handles returned HTTP 404, mapped to a `not_found` outcome that is distinguishable from a missing code. **No private account surfaced in 15 random real clipper handles** (private clippers are rare because their content must be public to earn); the helper captures `is_private`, and an Instagram bio is public metadata even on a private account, so a private account still returns its bio as a `resolved` outcome.

The bio fetch returns a **distinguishable** verdict by design:
* **resolved** — HTTP 200 + user object; `biography` is `""` for a real no-bio account (NOT an error).
* **not_found** — HTTP 404: the profile does not exist. Message points at the profile link, never says the code is missing.
* **transient** — 429 / 402 / 5xx / network / bad-JSON / no-key: a provider hiccup → `transient:true` → the POST refunds the rate slot and the clipper sees a "please try again" message. **A fetch failure is never a missing-code verdict** (BL-518/521).
