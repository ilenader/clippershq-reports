## The money-file changes (scrutinised, printed, confined)
Two money files changed by design; the other four plus campaign-era.ts are byte-identical by blob OID on BOTH refs.

**balance.ts** `f8499e18 -> e887f80a` (in `getCampaignBudgetStatus`, a read-only status function):
- Adds `platformFeePctDecimal` to the existing select (additive read).
- Computes `{ ghostFee, realBudget } = realBudgetFromFee(budget, platformFeePctDecimal)`. A NULL fee returns `{ ghostFee: 0, realBudget: budget }`.
- `spent = round2(spent + ghostFee)` (the ghost is spent the instant the campaign starts; 0 for a fee-less campaign, so spent/remaining/isOverBudget are unchanged).
- `clipperPoolCap = round2((1 - s) * realBudget)` and `ownerReserveCap = round2(s * realBudget)` (was `campaign.budget`). realBudget == budget when the fee is NULL, so byte-identical.
- Touches NO earnings, bonus, payout, status, or cadence line. Confined to the pool-cap derivation.

**tracking.ts** `b311323c -> 847dcf70` (in `processTrackingJob`, the budget-pause block):
- Adds `platformFeePctDecimal` to the tx.campaign select; computes `txRealBudget` via realBudgetFromFee.
- Three cap sites swap `txCampaign.budget -> txRealBudget`: the guarantee `clipperPoolCap` + `ownerReserveCap`, the pause-check `clipperPoolCap`, and the pauseReason owner-reserve string.
- txRealBudget == txCampaign.budget for every fee-less campaign, so byte-identical arithmetic. The clip's `newEarnings` (views x CPM), `writeClipEarnings`, status, and cadence are untouched. The only change for a fee campaign is that it auto-pauses when the clipper pool of REAL money is exhausted (a lower cap), which is the intended fix.

**platform-fee.ts** (new, the single source of the fee math): pure `Prisma.Decimal` / decimal.js, NEVER float. `ghost = budget.mul(pct).div(100).toDecimalPlaces(2, HALF_UP)`, `real = budget.minus(ghost)`. NULL / non-positive / non-finite / >=100% fee / budget<=0 all return `{ ghostFee: 0, realBudget: rawBudget }`. So a fee-less campaign is byte-identical to pre-BL-630 by construction. The $5,000 / 11% case: ghost $550, realBudget $4,450; the closed leak is the clipper-pool share of the ghost, (1 - 0.67) x $550 = **$368.50** per $5,000 campaign.

Neither diff exceeds "derive the pool cap from realBudget", so no halt was warranted.
