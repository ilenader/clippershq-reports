### The query for the owner to watch it himself (run any time)
```sql
-- Connections today + the last few, with exact UTC times; compare to the ~5/day norm.
SELECT date_trunc('day',"verifiedAt")::text AS day, count(*) AS connected
FROM clip_accounts WHERE "verifiedAt" > now() - interval '14 days'
GROUP BY 1 ORDER BY 1 DESC;

-- Are attempts FAILING (a defect) or just not happening (traffic)?
SELECT "lastVerifySource", "lastVerifyFoundInBio", status, count(*)
FROM clip_accounts WHERE "lastVerifyAt" > now() - interval '24 hours'
GROUP BY 1,2,3 ORDER BY 4 DESC;

-- Anyone stuck mid-connection right now?
SELECT count(*) AS pending_created_24h
FROM clip_accounts
WHERE status='PENDING' AND "deletedByUser"=false AND "createdAt" > now() - interval '24 hours';
```
A day of zero connections is only worth worrying about if it is paired with NEW pending accounts whose `lastVerifyFoundInBio` is false or a spike in `transient`. Neither is present now.
