## The owner's hard lines, and where each is enforced

• AUTO-REJECT only machine-CERTAIN: only REQUIRED_SOUND (both ids present +
  mismatch) and REQUIRED_CAPTION_TOKENS (a literal token absent) can ever return
  `would_reject` (`src/lib/campaign-rules.ts` evaluateRuleShadow).
• Everything visual/logo + borderline RANKS: LOGO_PRESENT_THROUGHOUT returns
  `rank_review`, never rejects.
• FAIL OPEN: missing caption / null sound id / empty params / unparseable
  rulesJson all return `cannot_evaluate` and never set `wouldReject`.
• SHIPS OFF: the evaluator is called ONLY to `console.log('[RULES-SHADOW] ...')`
  inside `after()`; it writes nothing to any clip.

## PART 1 — schema + form

Column: `Campaign.rulesJson Json?` added additively. Applied to the DB with
`ALTER TABLE campaigns ADD COLUMN IF NOT EXISTS "rulesJson" JSONB;` (via
run-schema-sql.js) and `prisma/schema.prisma` + `npx prisma generate` ONLY (no
`prisma migrate`). NULL on every existing campaign. `requirements String? @db.Text`
stays the BYTE-IDENTICAL clipper-facing source of truth; rulesJson is the machine
twin beneath it.

Form (`src/app/(app)/admin/campaigns/page.tsx`): the old
`form.requirementsList.filter(...).join("\n")` is preserved EXACTLY for
`requirements` (clippers see the same text). Added an index-aligned
`ruleTypesList` and a per-rule TYPE `<select>` beside each requirement Input, 7
types (default HUMAN_ONLY): Human review, Required caption text, Required sound,
Logo present (rank), Posting window, Video length, Edit quality. On save the form
builds `rulesJson` from the aligned lists (tokens for REQUIRED_CAPTION_TOKENS are
extracted LITERALLY from the text, never invented; other types carry empty params
so the evaluator fails open). `rulesJson` is persisted through POST
(`/api/campaigns`) and PATCH (`/api/campaigns/[id]`, whitelisted like `sounds`).

No-rules campaigns still work: `parseRulesJson(NULL)` returns `[]`, openEdit
defaults every row to HUMAN_ONLY, and a campaign saved with no structured rules
sends `rulesJson: null`. Old campaigns are unaffected until an owner opens + saves.

Accessibility: reviewed by the accessibility-lead before the edit. Applied: unique
`aria-label` per select (`Requirement N type`) and per remove button
(`Remove requirement N`), native `<select>` (free keyboard + mobile picker),
`Trash2 aria-hidden`, focus restored by computed id after re-render (rows keyed by
index), 44px touch targets, `text-base lg:text-sm` (no iOS zoom), and a
`flex-col sm:flex-row` reflow with the grow/shrink on a wrapper around `Input`

## PART 3 — the free checks, in SHADOW (proven, no rejection)

`src/lib/campaign-rules.ts` is pure (no DB, no network, no money-file import). The
submit path already fetches the provider payload for freshness and carries it as
`ClipStats.raw` (TikTok mapper `raw: item`, IG mappers too; YouTube has none ->
fail open). `extractClipMetadata(raw)` tolerantly reads caption / sound id /
hashtags from that SAME payload at ZERO extra vendor cost.

Wiring: `/api/clips` POST captures `fetchedRawMeta = stats.raw`, adds `rulesJson`
to the campaign select, and in an `after()` block (post-response, best-effort,
wrapped in try/catch) runs `evaluateRulesShadow` and logs `[RULES-SHADOW] clip=…
realOutcome=PENDING wouldReject=… wouldRejectRules=[…] rankRules=[…] evaluated=…
failedOpen=…`. It writes NOTHING to clip status. A throw can never reach the
submit.

Proven against the REAL backfilled GainzAlgo rule (`#gainzalgo`, auto_reject):
```
caption HAS #GainzAlgo  -> would_pass       (case-insensitive NFKC match)
caption MISSING token   -> would_reject     "token(s) literally absent: #gainzalgo"
caption NULL            -> cannot_evaluate  wouldReject=false, failedOpen=8 (fail open)
```
The other 7 GainzAlgo rules are human_only and always fail open.

Would-reject rate on live data: measured GOING FORWARD from the `[RULES-SHADOW]`
logs on new submissions. A retrospective rate is NOT computable here because no
caption is persisted on historical clips (`ClipStats` stores only counts +
createdAt; caption rides only on the transient submit-time `.raw`) and this round
took no new vendor calls to re-fetch. The ceiling is bounded by the 8 auto_reject
rules across 5 campaigns; a clip is flagged only when a clipper on one of those
campaigns literally omits the required token.

## PART 4 — the logo check, rank-only (design wired, vision deferred)

LOGO_PRESENT_THROUGHOUT is wired in the evaluator as `rank_review` — it NEVER
returns would_reject and never sets wouldReject; it surfaces the clip in the
`rankRules` list of the shadow log for the human logo-review lane. The ACTUAL
4-frame sampling (0/25/50/75%) + template/OCR vision is deferred to its own round,
stated plainly here. R-6 confirms the cover frame is already on Supabase
`clip-thumbnails/` (15/15 HTTP 200), so the cheap first step is a cover-frame
check; at ~77 clips/day the ranking cost is negligible (Haiku-tier ~$2.24/mo for
the full 2,301-clip month per R-6). This round ships the rank hook + the rule type
+ the shadow ranking; no logo rule can reject anything.
