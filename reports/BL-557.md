## THE REAL MECHANISM — and it is not a re-derived set

**The gate is correct. The verdict feeding it is not.**

`lamatok.ts:131-136` decides `gone`:

```ts
const verdict = urlClass === "photo" ? "carousel"
  : res.status === 404 && urlClass === "video" && !isRegionBlock ? "gone"
  : "transient";
```

**A `gone` verdict REQUIRES `urlClass === "video"`.** And `classifyTiktokUrl`
(`hikerapi.ts:156-160`):

```ts
if (/tiktok\.com\/[^/]+\/video\//i.test(url)) return "video";
if (/tiktok\.com\/[^/]+\/photo\//i.test(url)) return "photo";
return "unknown";                       // <-- every vm./vt. short link lands here
```

**A short link can never be marked gone, no matter how dead it is.** Proven live on a dead short link
clockworks was actually fed:

```
--- https://vt.tiktok.com/ZSCMwWdUH/
  LamaTok HTTP 404  body: {"detail":"Media Not found","exc_type":"MediaNotFound"}
  urlClass=unknown  http=404  regionBlock=false
  => VERDICT: transient   <-- 404 MediaNotFound but NOT marked gone, because
                              urlClass !== video. ESCALATES TO CLOCKWORKS.
```

**That is a definitive 404 being downgraded to "maybe, try Apify".**

**And it is the majority of the fleet.** Live tracked TikTok clips:

**69% of the TikTok fleet is structurally incapable of receiving a gone verdict.**

## PART 3 — the 13:14:46 run, forensically. BL-555 was right.

**Input:** exactly one URL, `https://vt.tiktok.com/ZSXPCSW3W/`. **Cost $0.0040.**

* **Is it alive now?** **Yes.** LamaTok returns HTTP 200 with the full media object,
  id `7663484330550791432`.
* **Did clockworks return a real number?** **Yes**, the same id, with a real view count.
* **Was LamaTok's failure transient or a 404?** **Transient, and the reason is decisive:**
  `createTimeISO: 2026-07-17T13:13:15Z`. **The video was 92 seconds old when clockworks ran.**
  LamaTok's index had not caught up to a brand-new post.
* **Was BL-554's 5xx retry live, and would it have caught this?** The retry shipped in the same merge
  that deployed at ~13:10, so it was live. **It would not have helped.** The retry fires on a 5xx or a
  network error. An unindexed brand-new video is not a 5xx: LamaTok answers, it simply does not have
  the post yet. Retrying 600ms later would almost certainly have missed again.

**So this run is a genuine rescue of a live clip, exactly as BL-555 said.** A fresh submit is the one
case where clockworks earns its fee. **BL-555's error was not the claim, it was the sample size: it
generalised from one run to the whole actor, and the whole actor is 97% waste.**

## PART 5 — the verdict

**Clockworks is NOT a genuine safety net today. It is 77% the Instagram bug wearing a TikTok label.**

**The dollar split, this run-log page:**

| | $ | share |
|---|---|---|
| **Confirming a deleted video is deleted** | **$0.3089** | **76.7%** |
| Slideshow capability LamaTok genuinely lacks (legitimate) | $0.0830 | 20.6% |
| **Rescuing a real live clip** | **$0.0111** | **2.8%** |

**But do NOT kill it.** BL-543's rule holds and it is not negotiable: an unresolvable clip must return
**NULL, never 0**, or tracking zeroes a clipper's views and freezes their earnings. Clockworks is what
catches the 92-second-old fresh submit (PART 3) and the slideshows. **Both are real. Killing the actor
would break them.** The waste is not that clockworks exists; it is that it is asked the wrong
questions.

### The fix, precise

**Not one line this time, but nearly.** BL-554's Instagram fix removed a duplicate derivation. This is
different: **the gate is correct and the verdict feeding it is too narrow.**

**The fix is in `lamatok.ts:131-136`: stop requiring `urlClass === "video"` for a `gone` verdict.**
A LamaTok **404 with `exc_type: "MediaNotFound"` and no region-block marker is definitive regardless of
URL shape.** The `urlClass === "video"` guard was written to be conservative (BL-262's comment says so:
*"CONSERVATIVE guards so we never drop a readable clip"*), and against canonical URLs it is. Against a
fleet that is **69% short links** it silently disables the skip for most of the platform.

Two safe options, in order of preference:

1. **Widen the verdict:** treat `404 + MediaNotFound + !isRegionBlock` as `gone` for `video` **and
   `unknown`**, keeping `photo` on `carousel` so the slideshow fallback survives untouched. This is the
   smallest change and it directly matches the evidence: every dead short link I probed returned exactly
   that body.
2. **Or resolve the short link first** so `classifyTiktokUrl` can see a real `/video/` URL. More
   correct, more moving parts, and it needs a redirect fetch per clip.

**What it would save:** **$0.3089 of $0.4030 on this page, i.e. 77% of clockworks.** Extrapolated at
today's rate that is roughly **$0.30 per day, about $9/month**, on top of the $1.35/day BL-554 already
killed. **The residual would be the slideshow gap plus fresh submits, which is the safety net actually
doing its job.**

**Caveat, stated plainly:** widening the verdict makes the skip fire on more clips, and a skip is
irreversible for that tick (the clip returns null and keeps its last-known value, per BL-543). If
LamaTok ever 404s a *live* short link, that clip would stop updating until it recovers. I saw no
evidence of that (every 404 I probed was genuinely dead, and clockworks independently agreed 88 times),
but **the fix deserves its own round with a shadow proof: log what WOULD be skipped for one day before
skipping it.** That is exactly the discipline BL-550 used, and it is cheap.
